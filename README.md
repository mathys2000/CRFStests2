# CRFStests2
Python notebooks for reading CRFS time-data csv files

The goal is to understand the structure of the csv files generated by the CRFS Mission Manager and to extract relevant data such as time stamps and I/Q data samples.

Another goal is to figure out how to develop Python code in Jupyter Lab while still using Git as a version control system. The approach taken here is to convert .ipynb files (which have too much metadata) to .py:percent files using Jupytext. Only the .py files are then included in the Git repository.

To set up syncing for a specific file use (in project root folder):
jupytext --set-formats ipynb,py:percent ReadIQ_csv_001.ipynb
To syn the paired file:
jupytext --sync ReadIQ_csv_001.py
